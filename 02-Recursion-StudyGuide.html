Turing Machines

A mathematical model of computation (Alan Turing, 1936)

What is a Turing Machine?

  • A mathematical model of computation
  • Invented by Alan Turing, a British mathematician, in 1936
  • The challenge: create a "mechanical procedure" to answer any mathematical statement as "yes" or "no"
  • Turing imagined a machine that could perform any conceivable mathematical calculation, given enough time and instructions

Components of a Turing Machine

Tape

An infinite strip divided into cells, each holding a symbol (0, 1, or blank). The machine's memory.

Head

A read/write head that sits on one cell at a time. It can read the symbol, write a new symbol, and move left or right.

State Register

The machine's current state (e.g., q_flip, q_halt). Determines which rule to follow.

Transition Rules

A table of rules: given the current state + symbol, write a symbol, move the head, and change state.

Rule Format: CurrentState — Read/Write, Direction — NextState
Example: q_flip — 0/1, R — q_flip means "In state q_flip, if you read 0, write 1, move Right, stay in q_flip."

The Flipper Problem

A simple Turing Machine that flips all 0s to 1s

The Setup

  • Tape: 0 0 0 (followed by blank spaces forever)
  • Starting State: q_flip (the "working" state)

The Rules

Rule 1: q_flip — 0/1, R — q_flip   (read 0, write 1, move right, stay in q_flip)
Rule 2: q_flip — B/B, R — q_halt   (read blank, keep blank, move right, halt)
Rule 3: q_flip — 1/1, R — q_flip   (read 1, keep 1, move right, stay in q_flip)
The Missing Rule: Rule 3 was initially missing from the problem! Without it, the machine wouldn't know what to do if it encountered a 1 already on the tape. This teaches us: every possible input must have a rule.

Church-Turing Thesis

The fundamental claim about the limits of computation

The Thesis: Anything that is computable (calculation, logic, rendering graphics, running AI) can be done by a Turing Machine.

Key Implications

1. Universality

Your laptop, phone, and a smart fridge are all mathematically equivalent to a Turing Machine. They are just faster.

2. Limits of Computation

If we can prove a Turing Machine cannot solve a problem, then no physical computer will ever solve it either.

Why It Matters

  • Gave us the blueprint of the modern general-purpose computer
  • Proved that some problems are undecidable (no algorithm can solve them)
  • Turing is widely considered the father of theoretical computer science and AI

The Halting Problem

Undecidable: A machine can never predict if an arbitrary program will eventually stop (halt) or keep running forever. This is the Halting Problem — it has been mathematically proven to be unsolvable.

Connection to Recursion

The Church-Turing Thesis tells us: anything computable by a Turing Machine can be computed using recursive functions. This is why recursion is a fundamental concept — it's mathematically equivalent to what any computer can do.

Factorial (Recursion)

The classic example: n! = n × (n-1) × ... × 2 × 1

The Problem

5! = 1 × 2 × 3 × 4 × 5 = 120

Recursive Solution

C# static int Factorial(int n) { Console.WriteLine($"Calculating factorial of {n}"); int result; if (n == 1) { result = 1; // Base case } else { int previousFactorial = Factorial(n - 1); // Recursive step result = previousFactorial * n; } Console.WriteLine($"Factorial of {n} is {result}"); return result; }
Two Parts:
  • Base Case (n == 1): Returns 1 directly — stops the recursion
  • Recursive Step: Calls Factorial(n - 1) then multiplies by n

Formal Definition of Recursion

"Reduction to Simplicity"

Recursion is a technique where a function solves a problem by calling itself with a simpler version of the same problem, until it reaches a case simple enough to solve directly.

Two Required Parts

1. The Base Case (Halting State)

The simplest version of the problem that can be answered directly without recursion. This is what stops the recursion from going forever.

if (n == 1) return 1;

2. The Recursive Step (Reduction)

The function calls itself with a smaller / simpler input, making progress toward the base case.

return n * Factorial(n - 1);

Without a Base Case: The recursion never stops, causing a StackOverflowException. Without reduction toward the base case, same problem — infinite loop. Both parts are essential.

How to Think About Recursion

  1. Trust the function: Assume the recursive call returns the correct answer for the smaller problem
  2. Build from there: Use that answer to construct the answer for the current problem
  3. Define the base: Identify the simplest case where you know the answer immediately

The Lazy Climber Problem

How many distinct ways to climb N stairs taking 1 or 2 steps at a time?

The Problem

You are at the bottom of a staircase with N steps. You can climb either 1 step or 2 steps at a time. How many distinct ways can you reach the top?

Example: N = 3

  • 1-1-1 (three single steps)
  • 1-2 (one step, then two steps)
  • 2-1 (two steps, then one step)

Answer: 3 ways

The Key Insight

Ways(n) = Ways(n-1) + Ways(n-2)
To reach step N, you either came from step N-1 (took 1 step) or step N-2 (took 2 steps). So the total ways equals the sum of ways to reach each of those steps. This is the Fibonacci sequence!

Recursive Solution

C# static int LazyClimberRecursive(int n) { int retVal; if (n <= 0) { retVal = 0; } else if (n == 1) { retVal = 1; // base case: 1 way to climb 1 step } else if (n == 2) { retVal = 2; // base case: 2 ways to climb 2 steps (1+1 or 2) } else { retVal = LazyClimberRecursive(n - 1) + LazyClimberRecursive(n - 2); } Console.WriteLine($"LazyClimberRecursive({n}) = {retVal}"); return retVal; }

The Palindrome Oracle

Recursively checking if a word reads the same forwards and backwards

The Problem

You can only see the first and last letter of a word. Determine if it's a palindrome by stripping off the ends.

Recursive Solution

C# static bool IsPalindrome(string s, int left, int right) { if (left >= right) return true; // Base case: 0 or 1 chars left if (s[left] != s[right]) return false; // Mismatch found return IsPalindrome(s, left + 1, right - 1); // Strip ends, recurse }

How It Works: "RACECAR"

2nd1 (A)5 (A)A == AMatch, continue
3rd2 (C)4 (C)C == CMatch, continue
4th3 (E)3 (E)left >= rightBase case: true!

Why Recursion Matters

More than just a programming trick — it's fundamental to computation

Church-Turing Connection

Anything computable by a Turing Machine can be computed using recursive functions. They are equivalent models of computation.

Native Language of Math

Recursion is the "native language" of mathematical models. Many definitions in math are inherently recursive (factorials, Fibonacci, tree structures).

Beyond Iteration

Some functions are so complex that they cannot be solved with iterations alone. Recursive solutions can express these naturally.

Key Takeaway: Recursion isn't just an alternative to loops — it's a fundamental computational concept that's mathematically equivalent to what any computer can do. Understanding recursion means understanding the nature of computation itself.

Tail-End Recursion

When the recursive call is the very last thing a function does

What is Tail Recursion?

A recursive function is tail-recursive if the recursive call is the last operation performed. There's no work left to do after the call returns.

Example: Print Numbers

C# - Tail Recursive static void PrintNumbers(int n) { if (n == 0) return; // Base case Console.WriteLine(n); PrintNumbers(n - 1); // Last thing - tail position! }

Equivalent Iterative Version

C# - Iterative with goto static void PrintNumbersIterative(int n) { begin: if (n == 0) return; Console.WriteLine(n); n = n - 1; goto begin; }
Key Insight: A tail-recursive function can be mechanically converted to a loop. The compiler can optimize it so it uses constant stack space instead of growing the call stack. This optimization is called tail-call optimization (TCO).

Tail vs Non-Tail Recursion

FeatureTail RecursiveNon-Tail Recursive
Recursive call positionLast operationMore work after the call
Stack usageCan be O(1) with TCOO(n) — grows with depth
Convertible to loop?Yes, mechanicallyNot directly
ExamplePrintNumbers(n-1)n * Factorial(n-1)
Why Factorial is NOT tail-recursive: In return n * Factorial(n-1), after the recursive call returns, we still need to multiply by n. That multiplication happens after the recursive call, so it's not in tail position.

Self-Reproduction & Quines

Can machines reproduce themselves? Programs that output their own source code.

Can Machines Self-Reproduce?

A self-reproducible program has two parts:

The Blueprint (Data)

A set of instructions that describes how to build something.

The Factory (Mechanism)

The part that takes a blueprint and actually builds the object.

Key Insight: To self-reproduce, the "Factory" needs to use the "Blueprint" twice:
  1. Once to build the output (using the blueprint as instructions)
  2. Once to copy the blueprint itself (using it as data)

Quine (named after Willard Van Orman Quine)

A quine is a program that takes no input and produces its own source code as output.

C /* The Blueprint: This string contains the source code, with %s where the string itself will go. */ char *s = "#include <stdio.h>%cint main() { %c char *s = %c%s%c;" " %c printf(s, 10, 10, 34, s, 34, 10, 10);" " %c return 0; %c}"; /* The Factory: Using the blueprint twice. */ printf(s, 10, 10, 34, s, 34, 10, 10);
Key Takeaway: Self-reproduction isn't magic — it's about a clever split between data and execution. As long as a program can treat its own code as data, it can replicate itself perfectly. This principle connects to biology (DNA = blueprint, cell = factory) and to theoretical computer science.
