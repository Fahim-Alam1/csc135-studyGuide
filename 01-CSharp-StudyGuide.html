String Interpolation

Tip: Use $"..." to embed expressions directly in strings. This is called string interpolation.

Complete Example

C# Console.Write("Enter your name: "); string name = Console.ReadLine() ?? string.Empty; Console.Write("Enter your favorite number: "); string numInput = Console.ReadLine() ?? "0"; if (!int.TryParse(numInput, out var favorite)) favorite = 0; decimal price = 12.34m; Console.WriteLine($"Hello, {name}! Your favorite number is {favorite}."); Console.WriteLine($"Formatted price: {price:C2}"); Console.WriteLine($"Uppercased name: {name.ToUpper()}");

Sample Output

Output Enter your name: Pradeep Enter your favorite number: 10 Hello, Pradeep! Your favorite number is 10. Formatted price: $12.34 Uppercased name: PRADEEP
Null Safety: Console.ReadLine() can return null. Use the ?? (null-coalescing) operator to provide a default value. Use int.TryParse instead of int.Parse to avoid exceptions on bad input.

Decisions

Control flow with if, else if, and else

if / else if / else

C# int a = 10, b = 3; Console.WriteLine($"a = {a}, b = {b}"); if (a > b) { Console.WriteLine("a is greater than b"); } else if (a == b) { Console.WriteLine("a equals b"); } else { Console.WriteLine("a is less than b"); }
Key Points:
  • Conditions must be bool expressions (no truthy/falsy like JavaScript)
  • Use == for equality comparison (not = which is assignment)
  • Braces {} are recommended even for single-line bodies

Loops (Iteration)

Repeating code with while and for loops

while Loop

C# int i = 1; while (i <= 3) { Console.WriteLine($"while loop iteration: {i}"); i++; }

for Loop — Iterating an Array

C# string[] fruits = { "apple", "banana", "cherry" }; for (int idx = 0; idx < fruits.Length; idx++) { Console.WriteLine($"for loop index {idx}: {fruits[idx]}"); }

for Loop — Sum First N Numbers

C# int sum = 0; for (int n = 1; n <= 5; n++) { sum += n; } // sum = 15 (1+2+3+4+5)
while vs for: Use while when you don't know how many iterations. Use for when you know the count or are iterating an index. Remember: .Length for arrays, .Count for lists.

Collections

Arrays (fixed-size) and Lists (dynamic)

Arrays (Fixed-Size)

C# // Initialize with values int[] numbers = new int[] { 10, 20, 30 }; Console.WriteLine($"array: {string.Join(", ", numbers)}"); // Create with 'new' and set values individually string[] names = new string[3]; names[0] = "Alice"; names[1] = "Bob"; names[2] = "Carol"; for (int i = 0; i < names.Length; i++) { Console.WriteLine($"names[{i}] = {names[i]}"); }
Arrays: Fixed size set at creation. Use .Length to get the size. Access elements with [index] (zero-based).

Lists (Dynamic)

C# var fruits = new List<string>() { "apple", "banana" }; fruits.Add("cherry"); // list grows dynamically for(int i = 0; i < fruits.Count; i++) { Console.WriteLine($"fruits[{i}] = {fruits[i]}"); } foreach(var fruit in fruits) { Console.WriteLine(fruit); } fruits.Remove("banana"); if (fruits.Count > 0) fruits.RemoveAt(0); fruits.Clear();
Lists vs Arrays: Lists grow/shrink dynamically. Use .Count (not .Length). Key methods: Add(), Remove(), RemoveAt(), Clear(). Also supports foreach loops.

Structures

Lightweight value types for grouping related data

Basic Struct

C# struct Point { public int X; public int Y; } var point = new Point { X = 3, Y = 4 }; Console.WriteLine($"Point: ({point.X}, {point.Y})"); // Structs are value types - assigning COPIES the value var copy = point; copy.X = 10; Console.WriteLine($"Original: ({point.X}, {point.Y})"); // (3, 4) Console.WriteLine($"Copy: ({copy.X}, {copy.Y})"); // (10, 4)
Critical Concept: Structs are value types. Assigning a struct to a new variable copies all data. Modifying the copy does NOT affect the original.

Struct with Methods

C# struct Rectangle { public int X; public int Y; public int Width; public int Height; public Rectangle(int x, int y, int width, int height) { X = x; Y = y; Width = width; Height = height; } public int Area() { return Width * Height; } } var rect = new Rectangle(0, 0, 5, 2); Console.WriteLine($"Rectangle area: {rect.Area()}"); // 10

Value vs Reference Types

Understanding how data is passed to methods

Pass by Value (Default)

C# static void ChangeByValue(int number) { number = 100; // Only the local copy is changed } int score = 10; ChangeByValue(score); Console.WriteLine(score); // Still 10!

Pass by Reference (ref)

C# static void ChangeByRef(ref int number) { number = 100; // The original memory location is updated } int score = 10; ChangeByRef(ref score); Console.WriteLine(score); // Now 100!
Exam Favorite: By default, value types (int, struct, etc.) are passed by value — the method gets a copy. Use the ref keyword on BOTH the method parameter AND the call site to pass by reference.

Pass by Value

Method gets a copy. Changes inside the method do NOT affect the original variable.

Pass by Reference

Method gets a reference to the original. Changes inside the method DO affect the original variable.

Generic Collection Objects

Type-safe collections using generics (<T>)

Common Generic Collections

CollectionDescriptionUse Case
List<T>Dynamic arrayOrdered collection that grows/shrinks
Dictionary<TKey, TValue>Key-value pairsFast lookup by key
Queue<T>First-In, First-OutProcessing items in order
Stack<T>Last-In, First-OutUndo operations, parsing
LinkedList<T>Doubly-linked listFrequent insertions/deletions
What is <T>? The T is a type parameter. It lets you specify what type the collection holds, e.g., List<string> holds strings, Queue<int> holds integers. This provides compile-time type safety.

Queue<T> — FIFO

Enqueue() adds to back
Dequeue() removes from front
Think: waiting in line

Stack<T> — LIFO

Push() adds to top
Pop() removes from top
Think: stack of plates
